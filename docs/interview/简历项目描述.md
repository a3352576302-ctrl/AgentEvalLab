# AgentEvalLab 简历项目描述

---

## 一句话项目描述

**AI Agent 自动化测试与评测平台——基于 pytest + YAML 驱动，通过 L1-L6 六层断言对 Agent 的工具调用轨迹、安全合规和 Token 成本进行全维度评测。**

---

## 三条简历 bullet

- 设计并实现 339 条 YAML 驱动的 Agent 评测 Benchmark，覆盖 functional/boundary/error/security 四类场景与 9 个业务 scene；通过 L1-L6 六层断言检查最终答案、工具序列、参数、轮次、安全和 Token 成本。
- 接入 DeepSeek 和 MiniMax 完成 226 条 LLM-only 全量评测，MiniMax 整体通过率 75.2%、DeepSeek 安全防御更优（75% vs 70%）；实现 17 种失败归因、双模型横向对比、HTML 报告和静态 Dashboard。
- 基于 pytest 构建自动化测试框架（382 passed + 226 skipped），支持 GitHub Actions CI、Docker 一键复现、FastAPI 服务化接口和 SQLite 结果持久化。

## AI Agent 测试开发岗位版本

**AgentEvalLab：AI Agent 自动化测试与评测平台** | 个人项目  
`Python · pytest · YAML · LLM Function Calling · FastAPI`

- 构建 339 条 YAML 用例的 Agent 评测 Benchmark，从最终结果、工具选择、参数格式、调用轮次、安全合规、Token 成本六个维度设计断言（L1-L6），覆盖单工具/多工具串联/安全对抗/异常降级等场景。
- 完成 DeepSeek 与 MiniMax 双模型 226 条全量评测，发现 MiniMax 知识查询更优（95% vs 30%）但安全防御弱于 DeepSeek（70% vs 75%）；设计 17 种失败归因分类器，区分模型能力缺陷、断言过严、Provider 错误和安全风险。
- 实现 6 种故障注入（超时/500/脏数据/空结果/权限/网络）验证 Agent 降级策略；增加 Prompt 注入、越权调用、敏感信息泄露等 52 条安全用例；支持 API 重试退避、中断续跑和 baseline 回归检测。

## 测试开发岗位版本

**AgentEvalLab：AI 自动化测试评测框架** | 个人项目  
`Python · pytest · YAML · SQLite · Docker`

- 使用 pytest + YAML 构建数据驱动测试框架，339 条用例覆盖功能/边界/异常/安全四类场景，支持按分类筛选、批量回归、失败定位和 HTML 报告。
- 设计 L1-L6 六层断言引擎（结果/工具序列/参数/轮次/安全/Token），支持 AND/OR 语义、参数归一化和拒答智能放行；17 种失败归因自动分类。
- 通过 GitHub Actions 持续集成（382 passed + 226 skipped），Docker 一键复现环境，FastAPI 提供任务提交与结果查询 API。

## 面试口述版本

"我在 GitHub 上做了一个项目叫 AgentEvalLab——AI Agent 的自动化测试平台。核心思路是：Agent 和普通软件不一样，输出不确定，只看最终答案不够。所以我设计了 L1 到 L6 六层断言——不仅检查答案对不对，还检查它调了什么工具、参数对不对、有没有兜圈子浪费 Token、有没有被 Prompt 注入。

用了 339 条 YAML 测试用例，对 DeepSeek 和 MiniMax 各跑了 226 条全量评测。发现 MiniMax 整体更强（75% 通过率），但 DeepSeek 在安全防御上更好。同一个框架、同一批用例，跑出了两个模型完全不同的画像。框架本身 382 条测试全绿，证明了断言逻辑的正确性。

技术上用了 pytest 参数化、YAML 驱动、FastAPI 后端、SQLite 存储、Docker 打包。整个项目从设计到实现到评测到报告，是我一个人完成的。代码在 GitHub 上，382 条测试全绿，README 里能直接看到双模型对比结果。"
